#### UX Feedback

根據目前訪談與補充回饋，針對 **投資 Talk 君 APP** 的 UI/UX 改版，整理以下關鍵問題與改善建議。整體方向應以 **減法、資訊聚焦、內容可讀性** 為核心，避免首頁承載過多低頻資訊。

### 0. 首頁改版重點

首頁應從「功能入口集合」調整為「重點資訊與內容更新中心」，讓使用者一打開 APP 就能快速掌握 Talk 君觀點、重要總經資訊與最新文章。

* **移除 Home 目前的 4 個 Tabs：** 首頁不需要再以 4 個分頁承載不同內容，容易讓資訊被拆散，也增加使用者判斷成本。建議改成單一路徑的垂直資訊流。
* **新增重點總經資訊：** 將使用者最關心的總經重點放在首頁明顯位置，例如 CPI、FOMC、非農、殖利率、美元指數、通膨與市場參與度等重要觀察。
* **移除「美股盤勢」區塊：** 此區塊與 Talk 君獨家內容或高價值判讀的關聯較弱，首頁不需要保留。
* **新增 News Feed：** 首頁應提供文章動態流，讓使用者可以看到 Talk 君分享的文章、社群小編整理的文章或其他重點內容更新。
* **News Feed 需要作者頭像：** 每則內容應清楚顯示發布者頭像、名稱、角色與發布時間，讓使用者能快速辨識是 Talk 君、社群小編或其他官方角色發布。
* **Banner 改為可左右滑動區塊：** 原本整張圖片式 Banner 建議改成 carousel，呈現更簡潔的插圖加文字形式。視覺風格可參考 Wise 類型的清爽插圖與短文案，而不是使用滿版圖片廣告感的 Banner。

建議首頁資訊排序：

1. 簡潔 carousel banner：插圖 + 重點文字 + 單一 CTA。
2. 重點總經資訊：今日/本週最重要事件與市場指標。
3. News Feed：Talk 君與社群小編的文章、觀點與更新。
4. 高價值功能入口：社團、內容專區或 AI Bot 等高頻功能。

### 1. 資訊架構與導航優化 (IA & Navigation)

目前的架構讓使用者感到冗餘且混亂，建議重新梳理主要路徑：

* **調整底部 Tab 列順序：** 使用者反映目前的五個 Tab 分佈不符合直覺。
* **個人中心位置調整：** 目前中間 Tab 放置「個人取向/Beta 計算機」，但一般 APP 習慣將個人中心放在最右側。
* **高頻功能前移：** 「社團」與「內容專區」是使用者最常使用的功能，目前卻被放在偏右或較不直覺的位置，建議依使用頻率調整至更顯眼的位置。
* **簡化社團層級：** 「社團」區塊層級過多，例如社團專區、聊天室、語音聊天室等，且不同名稱可能導向相同地方，導致使用者難以記憶路徑。
* **強化重要資訊能見度：** 使用者非常看重「市場參與度」與 CPI 等重大事件發布日，但這些資訊目前隱藏在宏觀頁面的中段，極難查找，應將其置頂或整合至首頁重點總經資訊。

### 2. 介面設計與視覺呈現 (UI Design)

* **降低深色模式限制：** APP 目前以黑色為背景，導致文字顏色選擇受限，多數資訊只能用白色呈現，難以區分狀態與層級。
* **Banner 視覺降噪：** 避免整張圖片式 Banner 帶來廣告感與視覺負擔，改用輕量插圖、短標題、輔助文字與清楚 CTA。
* **區分「已清倉」標的：** 對於持倉比例變為 0% 的股票，建議使用 gray out 灰階化處理，與活躍持倉做出視覺區隔，避免混淆。
* **優化影片標題顯示：** 目前影片標題過長且前綴過多重複資訊，例如「會員專屬」，導致在有限手機空間內無法看到核心標題內容。
* **影音播放 UI：** 使用者反映在直屏播放橫向影片時，上下留白過多且畫面過小，視覺體驗不佳。

### 3. 內容與社群體驗

* **建立文章型 News Feed：** 除影音內容外，首頁需要能持續展示文章、觀點、社群整理與重點摘要。
* **作者識別清楚：** 每則內容需要包含頭像、作者名稱、角色標籤與發布時間，降低使用者判斷資訊來源的成本。
* **社團過濾與搜尋功能：** 使用者希望能過濾雜訊，只看特定人物，例如 Talk 君或小助手的發言。
* **搜尋能力擴充：** 目前搜尋僅限於標的代碼，應增加關鍵字搜尋與特定用戶搜尋。
* **收藏功能 (Bookmark)：** 增加收藏夾功能，讓使用者能保存高品質貼文、文章或重要資訊。
* **內容形式多樣化：** 提供 AI 轉文字稿，方便使用者在通勤、旅遊或不便看影片的環境快速閱讀。
* **強化 AI Bot 可用性：** AI Bot 能有效縮短尋找資訊的時間，應維持易用性並確保可觸及足夠內容權限。

### 4. 投資資訊與持倉呈現

* **持倉細節透明化：** 使用者在意持倉變動的過程，例如是從 5% 減到 0%，還是一次賣完，建議在 UI 上呈現變動歷史或總報酬率。
* **總經資訊優先化：** 對使用者而言，重大總經事件與市場狀態比一般盤勢摘要更有價值。首頁應優先呈現事件、日期、指標與 Talk 君判讀，而非泛用型市場盤勢。

### 5. 效能與技術體驗改善

* **載入效能：** 使用者對「知識庫」的載入速度非常不滿，經常出現 10 秒以上白屏，UX 目標應設定在 3 秒內完成載入。
* **背景播放穩定性：** 語音直播或影片在手機鎖屏後會中斷，尤其影響開車通勤情境，應優化背景播放，確保鎖屏後聲音不中斷。
* **登入狀態維持：** 減少強制重新登入的頻率。

### 6. 跨平台與服務體驗

* **網頁版需求：** 手機螢幕對於大量文字閱讀太小，使用者強烈期待有網頁版，除了方便看長文，也能解決 Apple Store 訂閱管理限制。
* **客服管道轉換：** 使用者偏好 Email 客服勝過 APP 內 Chatbot，因為 Chatbot 缺乏通知機制，且自動回覆流程冗長重複。

### 總結建議

新版 UI 應以 **減法** 為核心，優先移除低價值或重複資訊，讓首頁聚焦於 Talk 君觀點、重點總經資訊與高品質內容更新。首頁不應像資訊看板塞滿模組，而應像一個清楚的內容與判讀入口：先看到重點，再進入社團、文章、影音或 AI 搜尋。
